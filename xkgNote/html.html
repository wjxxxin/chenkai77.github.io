基本标签

Doctype是作用于文档最顶部的文档声明，是告诉浏览器是以标准模式还是以怪异模式展示该页面。Doctype不存在或格式错误都会导致页面以怪异模式展示页面。(标准模式的页面排版和JS运作模式都是浏览器支持的最高标准，而怪异模式是向后兼容，模拟老浏览器模式行为，防止页面无法正常工作)

行内元素/块级元素/空元素

  1. 行内元素: a/img/span/b/strong/input/select/section
  2. 块级元素: div/p/table/ul/ol/li/h1-h6
  3. 空元素: br/hr/img/input/link/meta

h1~h6(标题)

  1. 用来显示标题
  2. h1最大，h6最小
  3. 一个网页中最好只有一个h1，作为最主要的标题

p（段落）

  1. 用来显示一个段落

pre（预格式化的文本）

  1. 被包围在 pre 元素中的文本通常会保留空格和换行符。而文本也会呈现为等宽字体。标签的一个常见应用就是用来表示计算机的源代码。

img（图片）

  1. 用来显示一个图片
  2. src :设置图片的URL
  3. width： 图片的宽度
  4. height：图片的高度
  5. title：设置图片在鼠标移上去时显示d额描述文本
  6. alt：占位文字（当图片显示失败的时候显示这个占位文字）

a（链接）

  1. 表示超链接标签
  2. href：表示要链接的地址(URL)。（前面添加http://协议符号） 可用于设置锚点链接，如下 
    <a href="#id"></a>
  3. target:在哪里打开
    1. _self:在当前页面打开（覆盖当前页面）
    2. _blank:在新页面打开
    3. _parent：在父页面打开
    4. _top：在顶层页面打开
    5. 某个iframe的name属性值：在某个iframe中打开

列表标签

  1. ul>li*X:无序列表标签
  2. ol>li*X:有序列表标签
  3. dl>dt+dd*X:项目列表标签

浏览器

浏览器主要分为两个部分:渲染引擎和JS引擎
  1. 渲染引擎：主要负责获取页面内容和排版渲染页面
  2. JS引擎：解析和执行JS来实现页面的动态效果，以及交互内容

常用的浏览器内核

  1. Trident内核：IE，MaxThon,TT,the World,360,搜狗浏览器等。[又称MSHTML]
  2. Gecko内核：Netscape6及以上版本，FF，MozillaSuite/SeaMonkey等
  3. Presto内核：Opera7及以上。 [Opera内核原为：Presto，现为：Blink；]
  4. Webkit内核：Safari，Chrome等 [Chrome的：Blink(Webkit的分支)]

浏览器是怎么对HTML5的离线储存资源进行管理和加载的？

  • 在线的情况下，浏览器发现 html 标签有 manifest 属性，它会请求 manifest 文件
  • 如果是第一次访问app，那么浏览器就会根据 manifest 文件的内容下载相应的资源并且进行离线存储
  • 如果已经访问过app且资源已经离线存储了，浏览器会对比新的 manifest 文件与旧的 manifest 文件，如果文件没有发生改变，就不做任何操作。如果文件改变了，那么就会重新下载文件中的资源并进行离线存储
  • 离线的情况下，浏览器就直接使用离线存储的资源。

    3. button：普通按钮

    4. radio：单选框

    11. url：网址

  3. placeholder：占位文字
  4. name：为了防止单选的时候，用户多项选择，只需给不同的input标签设置相同的name属性值即可，复选框 name属性不会限制用户多选。

音频 视频标签

video 视频标签

  1. src：视频路径（必选属性）
  2. controls：控制条，表示是否添加播放控件；（一般情况这个都是不开放给用户的）
  3. autoplay：是否自动播放，在谷歌浏览器中是不能自动播放的，在最新的火狐浏览器也是默认被禁用，需要手动打开
  4. width：宽度
  5. height：高度
  6. loop：设置视频是否循环播放；（广告肯定有这个属性）
  7. preload：预加载（当具有preload的时候，autoplay会相冲突，不起作用）
  8. poster：视频封面
  9. muted：静音
<video src="media/super.mp4" controls width="500" poster="images/yx.jpg" autoplay loop preload muted>
</video>

source：设置video标签的多个格式的视频。浏览器会优先播放支持的格式。使用的基本结构：video>source 相当于又是一个 ul>li结构；标签属性：src：视频路径；type：表示设置视频的格式

audio 视频标签

audio几乎拥有所有的video属性，除了3个可见属性：width，height，poster。

页面导入

页面导入样式时,使用link和@import的区别

  1. link属于XHTML标签,除了加载CSS之外还能用于定义RSS,@import是CSS提供的,只能用于加载CSS
  2. link加载的文件,在页面加载的时候,link文件会同时加载,而@import引入的CSS文件,是页面在加载完成后再加载的
  3. @import有兼容性问题,IE5以下的浏览器是无法识别的,而link无兼容性问题

iframe标签

iframe 元素会创建包含另外一个文档的内联框架（即行内框架）。(在页面嵌套iframe标签指定src)

<iframe src="demo_iframe_sandbox.html"></iframe>

Iframe的缺点

  1. Iframe会阻碍页面的onload事件
  2. 浏览器的搜索引擎一般读无法解读iframe页面,不利于SEO的搜索
  3. Iframe和主页面共享链接池,会影响页面的并行加载
  4. 使用js动态添加iframe src属性,可以避免一三问题

Canvas和Svg

Canvas绘图

Canvas是HTML5提供的一个新标签。Canvas是一个矩形区域的画布，通过使用Canvas相关API，可以绘制路径、矩形、圆形、字符以及图像等。

<canvas id="canvas" width="400" height="300"></canvas>

兼容性问题： IE9+，可以在标签中间设置提醒

<canvas id="canvas" width="400" height="300">当前浏览器不支持canvas，请升级浏览器!</canvas>

cancas绘制路径的时候是中间对齐，所以1px的路径线条看起有 2px ，颜色变淡了

  1. Canvas标签默认拥有300 * 150的宽高，可以通过设置边框来查看。
  2. Canvas标签支持鼠标右键保存图片。
  3. Canvas标签参考坐标以左上角为起始点（0，0）。
  1. getContext("2d")：获取Canvas上下文对象，提供了丰富的API来帮助我们绘制路径、圆弧和矩形等图案
  2. moveTo()：设置Canvas上下文绘制路径的起点，相当于设置画笔从哪个位置开始移动。
  3. lineTo()：设置Canvas上下文绘制路径的目标点，相当于设置画笔移动的目标位置。
  4. strokeStyle：设置描边的样式。
  5. stroke()：根据坐标点绘制线条。
  6. beginPath：开启路径的作用是将不同的绘制路径进行隔离；
  7. closePath：闭合路径会自动连接最开始和最后的点；
Canvas绘制矩形方法
  1. rect(x, y, width, height) ： 设置矩形的路径 --> stroke() | fill()
  2. strokeRect(x, y, width, height)： 绘制描边矩形(独立路径)
  3. fillRect(x, y, width, height) ： 描绘填充矩形(独立路径)
  4. clearRect(x, y, width, height) ： 擦除矩形区域

Canvas和Svg的区别

  1. svg绘制出来的每一个图形的元素都是独立的DOM节点,能够方便的绑定事件或用来修改。canvas输出的是一副画布
  2. svg输出的图形是矢量图形，后期可以修改参数来自由放大缩小，不会失真和锯齿。而canvas输出标量画布，就像一张图片一样，放大会失真或者锯齿

前端SEO

  1. 合理的title、description、keywords：搜索对着三项的权重逐个减小，title值强调中你的那即可，重要关键词出现不要超过2次，而且要靠前，不同页面title要有所不同；description把页面内容高度概括，长度合适，不可过分堆砌关键词，不同页面description有所不同；keywords列举出重要关键词即可
  2. 语义化的HTML代码，符合W3C规范：语义化代码让搜索引擎容易理解网页。
  3. 重要内容HTML代码放在最前：搜索引擎抓取HTML顺序是从上到下，有的搜索引擎对抓取长度有限制，保证重要内容一定会被抓取。
  4. 重要内容不要用js输出：爬虫不会执行JS获取内容
  5. 非装饰性图片必须加alt
  6. 少用iframe：搜索引擎不会抓取iframe中的内容
  7. 提高网站速度：网站速度是搜索引擎排序的一个重要指标

网页从输入网址到渲染完成经历的过程

  • 1、输入网址
  • 2、发送到DNS服务器，并获取域名对应的web服务器对应的IP地址
  • 3、与web服务器建立TCP连接
  • 4、浏览器向web服务器发送http请求
  • 5、web服务器响应请求，并返回一定url的数据（或错误信息，或重定向的新url地址）
  • 6、浏览器下载web服务器返回的数据及解析html源文件
  • 7、生成DOM树，解析css和js，渲染页面，直至显示完成
